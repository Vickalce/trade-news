What This App Will Do

Input

  • Live news from free or low-cost sources
  • Your watchlist and market context
  • Price and volume changes

Output

  • Priority alert: low, medium, high
  • Action label: Buy candidate, Sell candidate, Hold
  • Confidence score with plain-language reason
Important: this is a decision-support assistant, not auto-trading software in MVP.

How It Works (Simple Flow)

1. Collect News
2. Filter Relevance
3. Map to Sectors and Stocks
4. Check Live Price Reaction
5. Score and Alert

Every alert includes why it fired and what could invalidate it.

Phase 3: Score and Recommend ✅ Delivered

The system combines four parts into one final score.

Score Inputs

  • Relevance score
  • Live market reaction score
  • Similarity to past events
  • Source quality score

Decision Output

  • High + positive reaction: Buy candidate
  • High + negative reaction: Sell candidate
  • Mixed or weak evidence: Hold

What’s Next — v2 Candidates

The MVP is running. These three improvements were identified during live smoke testing as the highest-value next steps.

Schwab OAuth Flow

Replace the manual bearer token with a proper OAuth 2.0 auth code flow so the system refreshes tokens automatically without operator intervention.

Next sprint

Position Sizing

Replace the simple floor(notional ÷ price) formula with a risk-based model (ATR or Kelly fraction) tuned per symbol and confidence tier.

Next sprint

Event Deduplication

Prevent the same event from being re-scored across multiple pipeline runs using persistent hash tracking stored in the database.

Next sprint

Engineering Guidelines Going Forward

  1. Keep scoring weights configurable — tune after at least 20 live signals.
  2. Show a clear explanation for every recommendation.
  3. Focus on reducing false positives before adding new signal types.
  4. Run paper validation for each new feature before enabling on live orders.
  5. Only disable the kill switch intentionally, in writing, after reviewing paper metrics.
